# FRP 兜底说明（仅在校园网直连失败时使用）

仅当以下条件满足时启用：

1. 另一台校园网机器无法直接访问服务器校园网 IP。
2. 已确认不是应用监听或防火墙配置错误。

## 思路

1. 服务器端（内网）跑 `frpc`，把本地 `127.0.0.1:80` 反向映射到中继机 `frps`。
2. 外部访问走中继机端口，再由 nginx 反代到应用。
3. 继续保留访问控制（CIDR 白名单或 basic auth）。

## 文件

- `frpc.toml.template`：内网服务器客户端模板。

## 注意

- FRP 不是默认路径。
- 启用 FRP 后，仍需限制访问来源，避免直接公网暴露。
